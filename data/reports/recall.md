# Recall against SemEval/DWUG gold (gap G3)

Generated by `eval/alignment/recall.py` (reproducible, no network, no LLM).
Merged graph: 156 Turtle files from `examples/` + `data/`.

## Question

Of the SemEval-2020 Task 1 / DWUG target words the gold standard labels
**"changed"**, what fraction carry at least one `drift:DriftEvent` in the
WORD-DRIFT graph?

## DWUG German

DWUG German lemmas present in-repo: **32**.
The repo carries DWUG German **sense-cluster structure only** (usage
graphs lifted to `drift:Word`/`drift:Sense`); it does **not** carry the
DWUG/DURel binary-change *gold labels*. DWUG therefore contributes **0**
to the gold-changed denominator here. This is a fixture-coverage limit,
not a true negative: ingesting the DWUG German graded-change gold (Zenodo
14028509, task1/task2 label files) is the documented next step and would
let DWUG German contribute to recall.

## Method

1. **Gold-changed set (SemEval EN)** = union of:
   - `etl/fixtures/semeval_en_targets.tsv` rows with `binary_change == 1`;
   - words carrying a `drift:DriftEvent` in `data/real/semeval_en.ttl` and
     `data/semeval.ttl` (the SemEval ingest, `etl/semeval_import.py`, emits
     a drift event only for `binary_change == 1` targets).
2. **Has-drift-event set** = every lemma that is the `drift:affectsWord` of
   some `drift:DriftEvent` in the merged `examples/` + `data/` graph,
   resolved via `drift:writtenForm` (preferred) or the IRI slug with the
   `word-<corpus>-` prefix and any POS suffix (`-nn`/`-vb`/...) stripped.
3. Recall = |gold-changed AND has-drift-event| / |gold-changed|.

## Honest caveats

- **This is coverage, not detection skill.** For the SemEval backbone the
  drift events are *ingested from* the gold binary labels, so a high
  SemEval recall is expected by construction and is a sanity check (does
  the ingest round-trip?), not evidence that WORD-DRIFT independently
  detects change. The scientifically novel layer is the *curated* causal
  hypotheses, which target a deliberately different (eponym/event/toponym-
  heavy) vocabulary that barely intersects the SemEval/DWUG target lists
  (see `data/reports/change-signal-alignment.md`, gap G2/G4).
- **DWUG German binary gold is not in-repo**, so German recall is not
  computed here. Stated as a limit, not hidden.
- Any unmatched gold-changed word indicates a SemEval target that lost its
  drift event somewhere in the pipeline (a real recall miss), and is
  listed above for audit.
