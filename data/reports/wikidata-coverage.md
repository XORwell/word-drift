**Newly linked this pass: 76** high-confidence matches (`data/wikidata/trigger-links.ttl`).

The original target was >= 70%. We deliberately stopped at 56.3% because the
remaining 132 unresolved triggers have **no single high-confidence Wikidata
referent** (see the unresolved list and reasons below). A wrong QID is worse
than an unlinked trigger, so coverage was traded for correctness: every emitted
link is an exact/near-exact label match whose Wikidata entity kind (person /
event / work / organisation / place) is consistent with the trigger and, where
a date exists on both sides, whose inception / point-in-time / date-of-birth is
consistent with the trigger's `drift:eventDate`.

> **Caveat (not hand-verified).** QIDs were selected by the matching heuristic,
> not individually confirmed against Wikidata by a human beyond the scoring
> rules. A human spot-check is advisable before publication. The links most
> worth re-checking are the lower-confidence ones (conf < 1.0) and any whose
> Wikidata entity could plausibly have a same-name namesake.

## Method (summary)

1. Load `examples/` + `data/`, list every `drift:TriggerEvent` without
   `owl:sameAs`, with its label / description / `drift:eventDate`.
2. For each *curated* trigger, query the Wikidata `wbsearchentities` action API
   for a steering search string (the eponymous **person** for eponyms, the
   **event** for events, the **work** for literary triggers, the originating
   **company** or the **place**), then fetch P31 + date claims via
   `wbgetentities` and score candidates (label match, kind consistency,
   date consistency, Wikidata search prominence).
3. Accept only at confidence >= 0.80. Eponyms link the person; events link the
   event; etc. Mythological-figure eponyms, brand-vs-generic trademarks, and
   vague discourse neologisms are intentionally left unlinked (see below).
4. The WDQS SPARQL endpoint is rate-limited to ~1 req/min during the 2024 wdqs
   outage, so all entity metadata is pulled from the `wbgetentities` action API
   instead. Lookups are cached on disk (`.cache/wikidata/`); the script is
   re-runnable and polite (descriptive User-Agent, delay between requests).
